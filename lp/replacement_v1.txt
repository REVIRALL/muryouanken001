【v1.5 緊急追記：実測データで判明した致命的問題】

LMES管理画面 + 運営者ヒアリングから判明した真のファネル:
  Meta imp → click 107 → LPV 82 → LMES到達 13 → 友達追加 0（広告経由）
  ※ 友達追加の 2件 は既存の知人（りゅう君・面談とんだ人）

つまり、
  (A) LP→LMES遷移で 82 → 13（約84%が離脱）
  (B) LMES→LINE友だち追加で 13 → 0（100%が引き返し）

本当のボトルネックは「LP本体」ではなく「LMES中継ページとLINE追加画面の警戒感」。
このため v1.5 では、LMES中継をできるだけ回避し、
  LP → LINE公式アカウント友だち追加URL（直リンク）
の経路を推奨する方針を追加した（下記の「実装メモ」参照）。

■ セクション9: よくある質問（Q&A）
================================================================

Q1. これはMLM・ネットワークビジネスですか？
A1. いいえ。当プログラムは「インフラ案件の紹介代理店」であり、
    ピラミッド型組織・ダウンライン報酬・権利収入設計ではありません。
    純粋な成果報酬型です。

Q2. ノルマはありますか？
A2. ありません。1件でも紹介できれば報酬が発生し、
    最低ラインや期限のノルマは一切設定していません。

Q3. コミュニティの規模はどれくらい必要ですか？
A3. 目安として50名以上を推奨していますが、属性によってはそれ以下でも可能です。
    15分の面談で最適なご提案をいたします。

Q4. LINE登録したら、すぐに勧誘されますか？
A4. されません。まずは15分のオンライン面談でヒアリングを行い、
    あなたに合うかを一緒に判断します。合わなければ断っていただいて構いません。

Q5. 報酬の支払いサイクルは？
A5. 月末締め・翌月末支払い（銀行振込）が基本です。
    詳細はご契約時にご確認いただけます。

Q6. 副業でも始められますか？
A6. 可能です。平日夜・週末の数時間でも運営できる設計になっています。
    面談時に、時間の制約に合わせた進め方をご提案します。

Q7. 途中でやめることはできますか？
A7. いつでも可能です。契約期間の縛りはありません。

■ 実装メモ（開発者向け）
================================================================

【HTMLテキスト化の意図】
- Meta LP品質スコア（読めるテキストの分量・比率）改善
- アクセシビリティ・翻訳・SEO下地（noindexだが将来外す場合に備える）
- A/Bテストの低コスト化（コピー変更＝画像再制作から解放）

【CTA発火ロジック変更（index.html:77-89）】
現状:
  CTA click → fbq('track','Lead') → setTimeout 300ms → LMES URLへ遷移

変更後（v1.5、3段階計測に分離）:
  CTA click      → fbq('track','InitiateCheckout')   ← 興味を示した
  LINE追加完了   → CAPI経由で fbq('track','Lead')    ← 本当のLead
  15分面談予約   → CAPI経由で fbq('track','Schedule') ← 質の高いLead

実装ルート（優先順）:
  A. LMES webhookで友だち追加を検知 → サーバーで CAPI Lead イベント送信
  B. LINE Messaging API の follow イベント → 自前サーバー → CAPI送信
  C. LIFF を使って友だち追加後ページで fbq('track','Lead') + track_id渡し

これにより、Meta AIが「実際に追加した人」の属性で学習でき、
配信品質が中長期で改善する。

【LMES中継の扱い（v1.5の新方針）】
- LP の CTA ボタンは LINE 公式アカウント友だち追加URL に直接リンク
- LMES は「追加後のウェルカムメッセージ」内で案内誘導に使う
- つまりファネルを:
  [変更前]  LP → LMES → LINE追加 → Welcome
  [変更後]  LP → LINE追加 → Welcome(LMES)
  → 中継1段減で離脱が減る想定
