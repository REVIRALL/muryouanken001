# LP v2 設計書（無料案件代理店プログラム）

最終更新: 2026-04-25
旧LPの実績: 2日間 LPV 161 / CTAタッチ 3 / LINE追加 0 / 消化 ¥93,310
失敗診断: 第1層（98%がCTAに触れない）+ 第2層（タッチしてもLINE追加されない）の同時破綻

---

## 1. ターゲット再定義（旧設計の自己ねじれを解消）

### 旧の矛盾
- 真ターゲット: MLM代理店ネットワーク保有者
- でも Meta規約上 MLM はターゲット不可 → ブロード配信
- LP訴求: 「副収入」「全員稼げる」← MLM語彙
- 広告コピー: 「コミュニティ運営者・サロン主宰者」← 規約安全語彙
→ **誰にも刺さらない**設計

### 新ターゲット（規約安全 + 訴求一貫）
**主**: コミュニティ・オンラインサロン運営者 / コーチ・コンサルタント / インフルエンサー
- 自身の有料商材を持っていない、または客単価の天井に当たっている
- 会員/フォロワーリストを持っているが収益化に困っている
- 「無料で会員に提供できる価値」を探している

**副**（拾えれば拾う、メインにしない）: 既存のアフィリエイター、副業情報系運営者

### ペルソナ
- 30-50歳、男女ほぼ均等（実績では25-34女性が最安）
- 自分のコミュニティ 100-3,000人規模
- 既に商材販売経験あり、月商 50-300万円帯
- 「MLMっぽいやつは絶対やりたくない」が口癖

## 2. メッセージ階層

### コアメッセージ
> **「あなたの会員に喜ばれる、"無料で得する案件" を運営者のあなたへ。」**

旧LPの「全員が手出しゼロで稼げる」=自己利益訴求 → **会員への提供価値**を主軸に転換。
これにより「副業臭」「MLM臭」を構造的に消す。

### サブメッセージ階層
| 階層 | メッセージ | エビデンス |
|---|---|---|
| 信頼 | 業界8年・累計730社の運営実績 | 実績数値 |
| 差別化 | 広告主と直接契約だから報酬が高い | 中間マージン排除の図 |
| 摩擦解消 | 業務はリンク共有のみ、会員の初期費用ゼロ | 仕組み図 |
| 自分ごと化 | コミュニティ運営者の声 | voice 3-4本（質重視） |
| 最後の一押し | 15分で診断、合わなければ参加不要 | 低摩擦オファー |

## 3. オファー階段の再設計

### 旧（高摩擦・1段）
広告 → LP → LINE登録 → 個別チャットで日程調整 → **15分対人面談**
→ 1段ずつ脱落、最終CV率 **0%**

### 新（低摩擦から階段化・3段）

```
Step 1: 即時CV（LP内完結）
  ┌─ 「3問で適性チェック」（Y/N×3問のクリック式）
  └─ 結果画面で「あなたに合う案件タイプ: ◯◯」+ LINE誘導
  → CV計測点①: 診断完了

Step 2: LINE登録 + Bot自動応答
  ┌─ ウェルカムメッセージで案件サンプル3件を即送信
  ├─ Bot質問「面談を希望しますか? Y/N」
  └─ Y選択 → カレンダー予約画面（TimeRex/Spir）
  → CV計測点②: LINE追加完了

Step 3: 個別面談（必要な人のみ）
  └─ 15分オンライン面談
  → CV計測点③: 面談予約完了 / 面談実施完了
```

**Pixel送信**:
- Step 1完了 = `Lead`（low-quality, ボリューム）
- Step 2完了 = `CompleteRegistration`（mid-quality）
- Step 3予約 = `Schedule`（high-quality）

Meta側にはまず Step 1 = Lead で学習を走らせ、量を貯めてから Step 3最適化に切替。
学習フェーズで Lead 0 が続く現象を構造的に防止。

## 4. ファネル設計とCTA階段

### CTA配置（IA）
```
[FV]
  └─ 即時CTA（プライマリ）: "3問で適性をチェック" → Step 1診断モーダル起動
  └─ サブCTA: "LINEで相談する" → 旧フローも残す（ヘビー検討者向け）

[共感セクション]
  └─ ソフトCTA: "あなたの組織は当てはまる?" → 診断へスクロール

[仕組みセクション]
  └─ なし（読ませることに集中）

[3ステップ]
  └─ ソフトCTA: "今すぐ3問チェック" → 診断モーダル

[実績/voice]
  └─ ハードCTA: "LINEで案件サンプルを受け取る"

[Q&A]
  └─ 最終CTA: "3問で診断 → LINEで案件を見る"

[フッター]
  └─ 法令表記・問い合わせ
```

### CTA文言ポリシー
- 旧: 全4箇所「今すぐ申し込む」（ハード一律）
- 新: 段階的にハード化、「申し込む」を排除
  - L1: 「3問で適性をチェック」
  - L2: 「案件サンプルを受け取る」
  - L3: 「無料で15分相談する」
  - 「申込」「契約」「登録」は **不使用**（心理摩擦が高い）

## 7. インタラクション設計（3問診断）

### 質問設計（Y/N or 4択）
1. あなたが運営するコミュニティの規模は?
   - 〜100人 / 100-500人 / 500-3000人 / 3000人〜
2. 会員に提供している商材は?
   - 自社商材あり / 他社商材紹介 / なし / その他
3. 関心が一番強いのは?
   - 会員に喜ばれる / 運営費を稼ぐ / 会員のリテンション向上 / 全部

### 結果分岐
- 結果は**全員「適性あり」**として表示（脱落防止）
- 質問内容は **リード情報の事前取得** が真の目的（Lステップ側でセグメント分け）
- 結果画面で:
  - あなたに合う案件タイプ
  - 「LINEで案件サンプルを受け取る」CTA
  - 入力された回答は LINEのトーク冒頭に連携（任意・将来）

### 計測
- 各質問回答時に `dataLayer.push({event:'quiz_answer', q:1, a:'...'})`
- 完了時に Pixel `Lead` 発火
- 結果画面で LINE誘導 → クリック時に Pixel `CompleteRegistration` 発火

## 10. 計測・タグ設計

### Pixel発火表
| イベント | タイミング | 価値 |
|---|---|---|
| `PageView` | 全ページロード | base |
| `ViewContent` | 50%スクロール到達 | engagement |
| `Lead` | 3問診断完了 | low-quality CV |
| `CompleteRegistration` | LINE誘導CTAクリック | mid-quality CV |
| `Schedule` | 面談予約完了（LINE Bot連携・将来） | high-quality CV |

### 発火実装ポリシー
- `navigator.sendBeacon` を最優先（同期送信、ITP耐性）
- `fbq` 標準も並行（重複は eventID で排除）
- ページ遷移前の setTimeout を **削除**（ITPで落ちる）
- すべて eventID 付与 → 将来 CAPI 連携時に重複排除可能

### CAPI（Phase 2）
- Netlify Edge Function で `/track` エンドポイント設置
- 同一eventIDで Meta Graph API `/{pixel_id}/events` POST
- iOS Safari ITP 耐性 +30〜40%回復目標
